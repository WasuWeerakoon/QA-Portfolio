# Demo Website – Test Plan

## 1. Introduction

This Test Plan describes the strategy, scope, resources, schedule, and approach
for testing the **Automation Exercise Demo E-commerce Website**.

The goal of this testing is to ensure that the core e-commerce functionalities
work correctly and provide a reliable user experience.

---

## 2. Project Information

- **Website Name:** Automation Exercise
- **Application Type:** E-commerce Web Application
- **Application URL:** https://automationexercise.com/
- **Testing Type:** Manual Testing
- **Testing Purpose:** QA Portfolio Demonstration
- **Environment:** Public Demo Website

---

## 3. Objectives

- Verify core e-commerce functionalities
- Validate user flows across modules
- Identify functional and UI defects
- Ensure proper validation and error handling
- Demonstrate complete QA documentation

---

## 4. Scope of Testing

### 4.1 In Scope

- User Registration
- User Login
- Product Listing & Product Details
- Cart Management
- Checkout Process
- UI and validation testing
- Positive and negative test scenarios

### 4.2 Out of Scope

- Performance testing
- Security penetration testing
- Backend API testing
- Payment gateway real transactions
- Load and stress testing

---

## 5. Test Approach

### 5.1 Testing Types

- Manual Functional Testing
- UI Validation Testing
- Regression Testing (basic)
- Cross-browser Testing (basic)

### 5.2 Testing Levels

- Component Testing
- System Testing

---

## 6. Entry & Exit Criteria

### Entry Criteria

- Website is accessible
- Test environment is stable
- Test data is available
- Test cases are prepared

### Exit Criteria

- All planned test cases executed
- Test results documented
- Bugs reported where applicable
- Test summary report prepared

---
